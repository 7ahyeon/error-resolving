# MySQL Error 1093
- 원인
  - MySQL : Oracle과 다르게 UPDATE/DELETE 시 자기 테이블의 데이터를 바로 사용할 수 없음
- 해결
  - 서브쿼리를 추가하여 서브쿼리 결과를 임시테이블 데이터로 만들어 실행
- 예제
  - 에러
```SQL
UPDATE company
SET id = '7ahyeon'
WHERE name = (
  SELECT name
  FROM home
  WHERE role IS NOT NULL
); 
```
  - 작동
```SQL
UPDATE company
SET id = '7ahyeon'
WHERE name = (
  SELECT name
  FROM (
    SELECT name
    FROM home
    WHERE role IS NOT NULL
  ) tmp
);
```
